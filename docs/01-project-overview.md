# 01 项目概览

## 1. 项目目标

本项目是一个面向手游场景的后端服务工程，承担登录、玩家加载、副本进入和副本结算等核心能力。

当前建设目标：

- 建立可维护的服务端工程骨架
- 建立清晰的分层与建模规则
- 建立适合商业化演进的规范和流程

## 2. 系统边界

系统当前覆盖：

- 客户端接入
- 登录鉴权
- 会话管理
- 玩家数据加载
- 副本进入与结算
- MySQL 持久化
- Redis 缓存与锁

系统当前不覆盖：

- 支付
- 社交
- 排行榜
- 活动中心
- 大规模微服务治理

## 3. 核心服务

- `gateway_server`
  负责接入、会话恢复、路由、上游转发。

- `login_server`
  负责账号鉴权和会话创建。

- `player_server`
  负责玩家数据加载与缓存协同。

- `dungeon_server`
  负责副本进入、状态校验、奖励结算。

## 4. 项目阶段判断

项目当前处于“从演示型工程收敛为正式工程文档体系”的阶段。

意味着：

- 文档不再围绕面试讲稿组织
- 结构不再围绕临时说明组织
- 技术表达需要面向长期维护和多人协作

## 5. 核心术语

- 协议模型：面向网络收发的消息结构
- 上下文模型：面向链路与请求元数据的统一上下文
- 应用层：面向用例编排
- 领域层：面向业务规则
- 基础设施层：面向存储、网络、日志和配置实现
