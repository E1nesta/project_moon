# 开发任务清单（tasks）

## 基本信息
- 生成日期：2026-03-28
- 需求主题：复现一套面向手游的 C++/Linux/MySQL/Redis 后端系统，覆盖账号鉴权、角色数据管理、副本进入与结算、日志排查与联调能力

## 任务结构（用户自定义）
阶段一：搭建网关、登录服、游戏服、MySQL、Redis 的本地开发环境与基础通信框架。  
阶段二：实现账号鉴权、玩家登录、角色加载/保存、在线会话管理。  
阶段三：实现副本进入条件判断、体力扣减、战斗结算、首通奖励、掉落发放、重复领奖校验、非法结算拦截。  
阶段四：补齐日志、配置表、GM/测试指令、接口联调与回归测试。  
阶段五：整理部署、压测、监控与后续扩展路线。

## 推荐任务拆解（可按自定义结构调整）

### 任务 1：确定复现边界与工程骨架
- 目标：把首版范围收敛到登录、角色、单人副本三条主链路，避免系统过大。
- 子任务：
  - 明确服务划分、协议格式、错误码和目录结构。
  - 选定 C++ 标准、网络库、序列化方案、MySQL/Redis 客户端库。
  - 初始化工程骨架，拆出 `common`、`gateway`、`login_server`、`game_server` 四部分。
- 输出：工程目录、构建脚本、依赖说明、架构文档。
- 完成标准：新成员拉取代码后能成功编译空服务并启动基础进程。

### 任务 2：完成环境与基础设施
- 目标：搭建本地可运行的最小环境，为业务开发提供稳定底座。
- 子任务：
  - 使用 Docker Compose 或脚本启动 MySQL 与 Redis。
  - 建立统一日志模块、配置加载模块、连接池与缓存访问封装。
  - 实现基础消息分发、连接管理、会话上下文和测试客户端脚本。
- 输出：环境启动脚本、初始化 SQL、配置样例、基础日志。
- 完成标准：服务启动后能建立连接、读写 MySQL/Redis，并输出结构化日志。

### 任务 3：实现账号鉴权与登录流程
- 目标：跑通“客户端登录 -> 会话签发 -> 角色加载”的闭环。
- 子任务：
  - 设计账号表、角色表和会话缓存结构。
  - 实现账号校验、令牌生成、重复登录处理、在线状态登记。
  - 实现角色基础数据加载、缓存回填和离线保存接口。
- 输出：登录接口、角色加载接口、会话缓存规则、错误码说明。
- 完成标准：测试脚本可完成登录、挤下线、断线重登、角色加载与保存。

### 任务 4：实现副本进入与结算核心逻辑
- 目标：跑通“进入副本 -> 创建挑战 -> 提交结算 -> 发放奖励”的闭环。
- 子任务：
  - 建立副本配置、玩家副本进度、奖励流水和挑战记录数据模型。
  - 实现进入条件判断，包括等级、体力、开放条件、未结算记录校验。
  - 实现结算合法性校验，包括 `battle_id`、副本编号、星级、时间窗口和重复结算校验。
  - 使用 MySQL 事务完成体力变更、奖励发放、副本进度更新和流水写入。
- 输出：副本进入接口、结算接口、奖励计算模块、事务与幂等策略。
- 完成标准：能够稳定覆盖正常通关、首通、重复结算、非法结算、资源不足五类场景。

### 任务 5：补齐联调、测试与可观测性
- 目标：让系统不仅“能跑”，还“能查、能测、能定位问题”。
- 子任务：
  - 增加 GM 指令或测试入口，支持加体力、重置副本、查看角色资源。
  - 补齐关键链路日志，统一打印账号 ID、角色 ID、`battle_id`、错误码。
  - 编写接口联调脚本和回归测试清单，覆盖主流程与异常流程。
- 输出：联调手册、日志字段规范、回归测试用例。
- 完成标准：出现登录失败或结算异常时，能在 5 分钟内从日志定位到失败节点。

### 任务 6：整理部署与后续演进路线
- 目标：让项目具备作品集展示能力，并可顺畅进入下一轮扩展。
- 子任务：
  - 提供一键部署说明、服务启动顺序和样例演示数据。
  - 记录性能瓶颈点、后续可拆分服务和建议优化方向。
  - 规划第二阶段扩展模块，如邮件、任务、排行榜、匹配或战斗服拆分。
- 输出：README、部署说明、演示脚本、二期路线图。
- 完成标准：从空环境到跑通完整业务闭环的时间控制在 30 分钟内。
